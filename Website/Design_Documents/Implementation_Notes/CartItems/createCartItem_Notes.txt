We need the following pieces of data:
user_id -> request_params | Session
product_id -> request_params | Front end

createCartItem ->
* Check if the request body is empty, if so, throw an EmptyRequestBodyError.
* Check if the user exists If not, throw a ResourceNotFoundError. -> WHEN SESSION COMES, THEN LOGGED IN. 

* Check if the cart item already exists, if so, throw a DuplicateSubdocumentError.

NOTE: We DON'T have to check if the product exists, since here, the user is clicking on the "Add to Cart" button which adds it to the Cart.

* Get the user_id from req.params.

* Get the cart_item_id from createRandomString(6).

* Create a DEEP CLONE of the request body.

* Create a filter using {user_id: user_id}.

* Create a cart_item using {cart_item_id, ...request_body_deep_clone} 

* Use the findOneAndUpdate(filter, {$push: {CartItems: cart_item}).lean() API to create the CartItem.

* Get the result from findOneAndUpdate() and send it back to the user as JSON with status 200.
