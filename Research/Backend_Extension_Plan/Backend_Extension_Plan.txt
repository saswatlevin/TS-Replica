To record transactions made by customers, a new collection called "Transactions". Each document in this collection will have the following structure:
  {
  "transaction_id": "txn_a1b2c3d4e5",
  "order_id": "77zlcwaiyjge",
  "user_id": "7ycthivdrqvl",
  "docType": "TRANSACTION",
  "transaction_type": "Payment",
  "transaction_status": "Completed",
  "amount": 358.00,
  "currency": "USD",
  "payment_method": "Visa",
  "visa_transaction_id": "visa_ref_xyz789",
  "date_created_at": "2025-05-21T12:45:00Z"
}

* Note that the "trasaction_type" field can either be "Payment" or "Return".
* Note that the "transaction_status" field can either be "Pending", "Completed" or "Failed".
* Note that the "visa_transaction_id" field stores the reference ID returned by the Visa API, which links your internal record to Visa's system.

* We will use VISA's sandbox API to conduct mock transactions.

To record returns, we will add the "returned_quantity", "return_reason" and "return_status" fields to the order_items sub-document in the Order document.
{
  "order_item_id": "n5shxmxiyrgy",
  "product_id": "hya3y1kdhv4p",
  "sku": "9qjmn3ak0b",
  "order_item_name": "The Craftsman Shirt in Bark Plaid Linen",
  "order_item_price": 128,
  "order_item_quantity": 1,
  "order_item_image_uri": "...",
  "returned_quantity": 1,
  "return_reason": "Defect",
  "return_status": "Completed"
}

* Note that "returned_quantity" defaults to 0 for all existing and new order items.
* Note that "return_reason" can be "Defect", "Cancelled" or "None".
* Note that "return_status" can be "None", "Requested", "Approved", or "Completed".

We need to add a single "quantity_returned" field in the "product_items" sub-document in the Products collection.
{
  "sku": "9qjmn3ak0b",
  "upper_size_letter": "M",
  "total_stock": 410,
  "quantity_sold": 410,
  "quantity_returned": 3,
  "current_stock": 3
}

* Note that when a return is "Approved", the "current_stock" field will increment by "returned_quantity" and we will have to add a Transaction document to the Transaction collection with transaction_type: "Refund" and call the Visa reversefundstransactions endpoint to push the money back.
