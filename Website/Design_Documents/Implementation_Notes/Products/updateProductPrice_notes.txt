pdWe need the following pieces of data:
* product_id -> Request Body | From frontend

updateProductPrice ->
* Check if the request_body is empty or not, if it is, throw an EmptyRequestBody Error.
* Check if the product exists or not, if not, throw a ResourceNotFoundError.

* Get a DEEP CLONE of the request_body
* Create the filter using { product_id : product_id }.
* Check if the productPrice that you are about to update already exists in the database. -> If yes, throw DuplicateSubDocumentError.

* Check if the discount_code is not set to "None". If it isn't, then do the following:
  * Get the discount_percentage from the respective Product document.
  * Calculate the discount_amount.
  * Calculate the discounted_total.
  * Add these parameters to the update_object.

* Use the findOneAndUpdate() function with the filter, the DEEP CLONE of the request_body, {runValidators: true} and chain it with .lean() to update the product price.
* Get the result from findOneAndUpdate(). -> RESULT 1

* Get the updated product price, updated discount code, updated discount percentage and updated discounted total from the result.
* Store the updated discount code in res.locals.
* Store the updated product price in res.locals.
* Store the updated discount percentage in res.locals.
* Store the updated discounted total in res.locals.


* Call updateCartItemPrice().
* Get the result from updateCartItemPrice(). -> RESULT 2

* Create an array of RESULT 1 and RESULT 2.

* Send the array with RESULT 1 and RESULT 2 as res.json.
* Set the res.status to 200.

NOTE: 
* res.locals is specifically designed for this purpose — it is an object that persists for the lifetime of a single request-response cycle and is not exposed to the client.
* res.locals is the preferred choice over req.locals because:
  * It is an officially documented Express property explicitly intended for this use case
  * It is scoped to the current request-response cycle and is automatically garbage collected after the response is sent
  * It is never serialized or sent to the client.
